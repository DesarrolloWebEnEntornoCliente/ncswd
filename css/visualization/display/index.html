Visualization of HTML elements

Space and content of HTML elements

Attending to its content, HTML elements can be enclosed in two groups:

  • Replaced elements. Element content is outside of CSS format. For instance: img, input, select, textarea, br, hr.
  • Non-replaced element. The navigator replace its content inside a box generated by the element itself. For instance: span, p, table

Regarding the space they occupy, we have:

  • Inline elements:
    1. Don't generata a content block
    3. Don't start in a new line if it is not necessar
    4. Span just to the space needed to show its content
    5. They only can contain other inline elements
    6. Can appear inside other inline elements or block elements
    7. Some examples: a, abbr, acronym, b, basefont, bdo, big, br, cite, code, dfn, em, font, i ,img, input, kdb, label, q, s, samp, select, small, span, strike, strong, sub, sup, textarea, tt, u, var
  • Block elements(Boxes):
    1. Generate a content block that span to all parent's content by default. Can't have elements at their sides
    2. Start in a new line
    3. Span to all available space
    4. They can contain other block elements or inline elements
    5. They can't be inserted inside inline elements
    6. Replaced elements can be block elements but normally they aren't
    7. Some elements: address, blockquote, body, center, col, dd, dir, div, dl, dt, fieldset, form, frameset, h1, h2, h3, h4, h5, h6, hr, isindex, li, menu, noframes, noscript, ol, p, pre, table, tbody, td, tfoot, th, thead, tr, ul
    8. These elements are also considered block elements: dd, dt, frameset, li
  • Special cases:
    • li is a special case since it generates a bullet or number added to the element's content
    • The followin elements can be inline or block depending on the markup: button, del, iframe, ins, map, object, script
